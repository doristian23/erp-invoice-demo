Desktop ERP — Invoice Validation Rules

These validation rules are enforced by the desktop application when creating or editing an invoice document.

Rule # Field Validation Error Message (Desktop)
V1 Customer Name Must not be empty Customer name is required. Please select a customer from the registry.
V2 Issue Date Must not be empty Issue date is required. Please select a valid date.
V3 Due Date Must be after Issue Date Due date must be after the issue date. Please correct the payment terms.
V4 Amount Must be greater than zero Invoice amount must be greater than zero. Please check line items.
V5 Invoice Number Must match pattern XX/YYYY/NNNNN Invoice number format is invalid. Expected format: XX/YYYY/NNNNN (e.g., FV/2024/00001).
V6 Line Items At least one line item required At least one line item is required. Please add items to the invoice.

Edge Cases (Desktop Behavior)

  • If a customer is deleted from the registry while an invoice references them, the invoice shows a warning icon and blocks editing until the customer is reassigned.
  • Invoices with status "Cancelled" cannot be edited or deleted — only viewed and exported.
  • Duplicate invoice numbers within the same fiscal year are rejected at save time.
  • When the due date falls on a weekend or bank holiday, the system auto-adjusts to the next business day and shows an info message.
  • Zero-quantity line items are allowed (for credit notes) but the unit price must still be non-zero.
